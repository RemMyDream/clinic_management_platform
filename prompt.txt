Using the entire current project codebase (including both Backend and Frontend) together with the `skill.md` / `skill.hd` file in the repository, perform the following tasks:

1. Analyze the entire codebase
- Identify all major modules/features currently implemented in the project.
- Group them by domain or feature area for easier management.
- For each module, provide:
  - Main functionality
  - Related Backend components
  - Related Frontend components
  - Complexity level or priority

2. Create a `progress.md` file
This file will serve as the main project tracking document for the team.

Required structure:
- Project overview
- 2-week implementation timeline
- Module list
- Task assignment for 4 team members
- Detailed task breakdown
- Progress tracking
- Risks / blockers
- Completion checklist

3. Split the work among 4 people
- Each person should own complete feature/modules (including both Backend + Frontend related work).
- Balance workload fairly across all members.
- Prioritize feature/domain-based ownership instead of separating Backend and Frontend work.
- For each person, include:
  - Assigned modules
  - Detailed task list
  - Deadlines
  - Dependencies (if any)

4. Create a 2-week execution plan
Split the timeline into:
- Week 1
- Week 2

For each week, include:
- Tasks to complete
- Review milestones
- Integration/testing milestones
- High-priority deliverables

5. Design an easy-to-manage tracking format
Inside `progress.md`, use:
- Markdown tables
- Checklists (`- [ ]`)
- Status labels:
  - TODO
  - IN PROGRESS
  - REVIEW
  - DONE
  - BLOCKED

6. Output requirements
- Generate a complete `progress.md` file.
- The content should be detailed enough for the team to use directly for project management.
- Prioritize readability, maintainability, and daily update convenience.
- If information is missing from the codebase, make reasonable assumptions based on the current project structure.

Important notes:
- Do not only list modules — break them down into actionable engineering tasks that can actually be assigned.
- Predict dependencies between modules to minimize conflicts during parallel development.
- Organize the structure similarly to a real sprint planning + engineering tracking document.
